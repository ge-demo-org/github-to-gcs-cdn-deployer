Agent Description

Google ADK와 A2A 프로토콜로 구현된 백엔드에서 Google Maps API를 툴로 호출해 레스토랑 및 경로를 탐색하고, 클라이언트 환경(Gemini Enterprise)에 맞춰 A2UI v0.8 JSON 페이로드를 동적 반환하여 지도 위젯을 그리는 에이전트

Core Concept

What is A2UI?

Agent to UI Protocol (A2UI) - a2ui.org

A2UI는 에이전트 주도형 인터페이스(Agent-Driven Interfaces)를 위한 프로토콜입니다.

다양한 탐색적 접근 방식들

Text Only

UI 표준이 부족하여 유저 경험이 매끄럽지 못하고 일관성이 떨어집니다.

Front End Apps

풍부한 UI를 제공하지만, 프론트엔드 앱을 에이전트 툴로 매핑하는 방식은 무겁고 유연성이 떨어집니다.

UIs as Tools

UI 클라이언트 위젯을 에이전트 툴로 매핑하면 UI를 더 표준화되고 효율적으로 렌더링할 수 있으며 개발이 단순해집니다.

Technical Architecture

A2A 및 A2UI 기반으로 다중 환경을 지원하며, 보안과 효율성을 극대화한 엔터프라이즈급 풀스택 아키텍처

주요 사용 기술

Google ADK (Agent Development Kit)

AI 에이전트의 백엔드를 구축하기 위한 핵심 개발 키트입니다.

A2A Protocol (Agent-to-Agent)

에이전트와 클라이언트 간의 원활한 상호 운용성을 보장하는 JSON-RPC 기반의 통신 프로토콜입니다.

A2UI (Agent-driven UI)

텍스트 인터페이스를 넘어, 에이전트가 직접 UI 구성 요소(예: 지도, 식당 선택 드롭다운 등)를 렌더링하도록 지시하는 UI 스펙입니다.

Google Maps API 그라운딩

식당 위치 및 길 찾기 정보의 정확성을 높이기 위해 Google Maps 데이터를 기반으로 정보를 제공합니다.

System Flow

아키텍처 및 UI의 핵심 특징

route

동적 버전 라우팅

클라이언트가 보내는 요청 헤더(X-A2A-Extensions)를 분석하여 클라이언트 환경에 맞는 A2UI 버전을 내려줍니다.

layers

v0.8 (Gemini Enterprise 환경)

A2UI 헤더가 없을 때 폴백(fallback)으로 제공되며, 렌더링 시작(beginRendering)과 화면 업데이트(surfaceUpdate)를 통해 인라인으로 컴포넌트 데이터를 처리합니다.
